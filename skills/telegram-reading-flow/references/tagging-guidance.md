# Tagging guidance for reading notes

## Goal

Keep the note taxonomy compact, reusable, and browseable.

## Rules

- Reuse an existing topic tag unless a new tag would clearly organize multiple notes better.
- Keep mechanics tags separate from topic tags.
- Prefer 2 to 4 meaningful topic tags per note.
- Avoid host/domain tags unless the publisher or product is itself the subject.
- If a tag will only ever apply to one note, it is probably too narrow.

## Current mechanics tags

- `reading-log`
- `x-post`
- `article`
- `historical-backfill`

These are provenance/mechanics markers, not the main reader-facing taxonomy.

## Candidate topic families

Use these families as stable defaults when they fit:
- `agents`
- `ai-infra`
- `security`
- `developer-tools`
- `org-design`
- `llm-research`
- `systems`

Only expand this set when repeated notes clearly demand a new bucket.
